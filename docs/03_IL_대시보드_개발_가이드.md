# OPIC IL 학습 대시보드 개발 가이드

개편일: 2026-06-12

## 제품 원칙

이 페이지는 학습·열람 전용이다. 사용자는 웹에서 질문이나 답변을 편집하지 않고 마스터 에이전트에게 자연어로 전달한다. 마스터가 내용을 분류·번역하고 소스 데이터에 반영한 뒤 검증과 배포를 수행한다.

## 화면 구조

1. 상단 유형 버튼에서 묘사, 루틴, 경험, 비교, 문제해결, 롤플레이, 의견 중 하나를 선택한다.
2. 선택한 유형의 질문은 접지 않고 바로 표시한다.
3. 실제 시험에서 전달받은 질문은 `기출 질문`, 에이전트가 만든 연습 질문은 `예상 질문` 태그로 구분한다.
4. 예상 질문의 추천 답변은 별도 버튼으로 펼친다.
5. 내 답변 패턴, 필러, 메모장은 기본 접힘 상태다.
6. 60초 연습과 영어 TTS를 제공한다.

페이지에는 질문, 답변 패턴, 필러의 추가·수정·삭제 폼이나 버튼을 두지 않는다.

## 소스 데이터

- `sourceQuestions`: 질문 유형, 기출/예상 구분, 영문, 한국어 설명, 원본 기출 연결
- `sourceAnswerPatterns`: 사용자가 직접 말한 한국어 소재와 쉬운 영어 문장
- `sourceFillers`: 기능별 필러 표현
- `personalNotes`: 추천 답변과 분리된 개인 메모

콘텐츠의 기준 원천은 위 소스 배열이다. 과거 버전의 질문·답변·필러 localStorage 값은 읽지 않는다. 기존 메모 숨김 상태인 `opicDeletedMemoIds`만 호환 목적으로 읽는다.

## 마스터 운영

```text
#마스터 P07 오늘 시험에서 카페에서 겪은 문제를 물어봤어.
```

마스터는 실제 질문을 `actual`로 등록하고 유형을 결정한다. 이 질문을 바탕으로 만든 유사 문항은 `expected`로 등록한다.

```text
#마스터 P07 나는 주말마다 집 근처 카페에 가서 아이스 아메리카노를 마셔. 답변 소재로 정리해줘.
```

마스터는 한국어 원문을 보존하고 초보자가 말하기 쉬운 3~5개의 영어 문장으로 정리한다. 사용자가 말하지 않은 개인 사실은 만들지 않는다.

## 추천 답변

추천 답변은 `sourceAnswerPatterns`만 사용한다. `personalNotes`는 어떤 경우에도 추천 입력으로 사용하지 않는다. 등록된 답변 패턴이 없으면 마스터를 통해 소재를 추가하라는 안내만 표시한다.

## 검증 기준

- 유형 버튼 전환 시 선택 유형 질문만 표시된다.
- 질문은 기본 펼침 상태다.
- 답변 패턴, 필러, 메모는 기본 접힘이며 토글된다.
- 콘텐츠 편집 폼과 CRUD 버튼이 DOM에 없다.
- 60초 타이머가 `01:00`에서 시작한다.
- 데스크톱과 390x844 모바일에서 가로 넘침이 없다.
- 추천 답변에 메모 내용이 섞이지 않는다.
- JavaScript 구문 오류와 브라우저 콘솔 오류가 없다.
