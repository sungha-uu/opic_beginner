# OPIC IL 학습 대시보드 개발 가이드

작성일: 2026-06-04
P07 개편일: 2026-06-12

## 1. 제품 목표

영어 초보자가 시험에서 실제로 들은 질문을 잊지 않고 기록하고, 자신의 경험과 취향을 쉬운 영어 소재로 쌓아 반복 연습하는 정적 웹 대시보드다. 시험 결과를 보장하지 않으며, 짧고 발음하기 쉬운 문장을 안정적으로 말하는 데 집중한다.

P07부터 미리 만든 유형별 모범 스크립트를 제품의 중심으로 사용하지 않는다. 제품의 중심 데이터는 다음 두 가지다.

1. 실제 시험에서 들은 `기출 질문`과 이를 바탕으로 만든 `예상 질문`
2. 여러 질문에 재사용할 수 있는 사용자의 개인 `답변 패턴`

## 2. 화면 구조

1. `질문 패턴`: 기출/예상과 질문 유형을 구분해 CRUD한다.
2. `답변 패턴`: 한국어 원문과 쉬운 영어 정리문을 독립 소재로 CRUD한다.
3. `60초 연습`: 저장된 질문 중 하나를 무작위로 보여주고 종료 후 추천 답변을 연다.
4. `필러`: 기능별 쉬운 연결 표현을 CRUD한다.
5. `메모장`: 기존 개인 메모를 표시한다. 추천 답변 데이터와는 완전히 분리한다.

## 3. 저장소와 호환성

| 저장소 | localStorage 키 | 역할 |
|---|---|---|
| 질문 | `opicP07Questions` | 기출/예상 질문, 질문 유형, 기출 연결 관계 |
| 답변 패턴 | `opicP07AnswerPatterns` | 한국어 개인 원문과 쉬운 영어 정리문 |
| 필러 | `opicP07Fillers` | 연결·시간 벌기·예시·정정·마무리 표현 |
| 메모 삭제 상태 | `opicDeletedMemoIds` | 기존 메모 숨김 상태, 이전 버전 호환 |

각 P07 저장소는 첫 실행 때 초기 배열을 저장하고, 이후 추가·수정·삭제 때 전체 배열을 JSON으로 다시 저장한다. 기존 개인 메모는 코드의 `personalNotes`에 유지한다.

## 4. 질문 모델

```json
{
  "id": "question-uuid",
  "typeId": "experience",
  "source": "actual",
  "questionKo": "최근에 공원에 갔던 경험을 물어봤다.",
  "questionEn": "Tell me about the last time you went to a park.",
  "originQuestionId": "",
  "createdAt": "2026-06-12T00:00:00.000Z",
  "updatedAt": "2026-06-12T00:00:00.000Z"
}
```

- `source: actual`: 사용자가 시험에서 실제로 기억해 전달한 질문이다. UI에서 `기출 질문` 태그로 표시한다.
- `source: expected`: 연습을 위해 만든 질문이다. UI에서 `예상 질문` 태그로 표시한다.
- 예상 질문이 특정 기출에서 파생되면 `originQuestionId`에 기출 질문 ID를 넣는다.
- 유형은 `description`, `routine`, `experience`, `comparison`, `problem`, `roleplay`, `opinion`을 사용한다.

## 5. 답변 패턴 모델

```json
{
  "id": "answer-uuid",
  "title": "퇴근 후 공원 산책",
  "category": "routine",
  "ko": "저는 퇴근 후 집 근처 공원을 걷습니다.",
  "en": "I walk in a park near my home after work.\nThe park is quiet, so I like it.",
  "tags": ["공원", "산책", "퇴근 후"],
  "createdAt": "2026-06-12T00:00:00.000Z",
  "updatedAt": "2026-06-12T00:00:00.000Z"
}
```

답변 패턴은 질문 유형별 모범 답안이 아니다. 경험, 취향, 루틴, 장소, 사람, 문제와 해결 등 여러 질문에서 다시 쓸 수 있는 개인 소재다.

영어 정리 원칙:

- 한 줄에 한 문장을 적는다.
- 한 소재는 보통 3~5문장 이내로 정리한다.
- `and`, `but`, `because`, `so` 같은 쉬운 연결어를 우선한다.
- 사용자의 한국어 원문을 임의로 과장하지 않는다.
- 어려운 표현보다 발음하고 암기하기 쉬운 문장을 선택한다.

## 6. 추천 답변 데이터 흐름

1. 예상 질문의 `typeId`를 읽는다.
2. 질문 유형에 어울리는 답변 패턴 카테고리를 우선 선택한다.
3. 선택된 개인 소재의 영어 정리문을 답변 뼈대로 표시한다.
4. 질문 유형에 맞는 쉬운 문장 한 개를 추가한다.
5. UI에서 `내 소재`와 `질문 맞춤 추가 문장`을 서로 다른 태그로 표시한다.

추천 답변 함수는 `state.answerPatterns`만 읽는다. `personalNotes`와 `opicDeletedMemoIds`는 절대 추천 입력으로 사용하지 않는다. 메모는 참고 화면일 뿐이다.

## 7. 필러 모델

```json
{
  "id": "filler-uuid",
  "functionId": "thinking",
  "en": "Let me think.",
  "ko": "잠시 생각해볼게요.",
  "createdAt": "2026-06-12T00:00:00.000Z",
  "updatedAt": "2026-06-12T00:00:00.000Z"
}
```

기능은 다음처럼 나눈다.

| ID | 기능 |
|---|---|
| `connect` | 말 연결하기 |
| `thinking` | 생각할 시간 벌기 |
| `example` | 예시 시작하기 |
| `correct` | 정정하기 |
| `closing` | 마무리하기 |

## 8. `#마스터 P07` 운영

사용자가 자연어로 기억나는 질문을 전달하면 에이전트가 질문 모델에 맞춰 반영한다.

```text
#마스터 P07
기출 질문: 시험에서 카페에서 겪은 문제를 물어봤다.
영어 기억: Tell me about a problem you had at a cafe.
질문 유형: 문제해결
```

기출은 반드시 `actual`로 저장한다. 기출을 바탕으로 문장을 바꿔 만든 연습 질문은 `expected`로 저장하고 원본 기출 ID를 연결한다.

```text
#마스터 P07
답변 소재: 라떼보다 아이스 아메리카노를 좋아하고 주말에 집 근처 카페에 간다.
소재 종류: 취향
```

답변 소재는 한국어 원문을 보존하고, 초보자가 말하기 쉬운 영어 문장을 줄 단위로 정리한다. 영어 번역이 불확실하거나 개인 사실 확인이 필요하면 임의로 만들지 않고 사용자에게 확인한다.

## 9. 60초 연습

1. 질문 저장소 전체에서 질문 하나를 무작위 선택한다.
2. 기출/예상 태그와 질문 유형, 영어, 한국어를 표시한다.
3. 60초 동안 타이머를 진행한다.
4. 사용자가 종료하거나 시간이 끝나면 추천 답변을 표시한다.
5. 저장된 답변 패턴이 없으면 먼저 개인 소재를 추가하라는 안내를 표시한다.

## 10. 검증 기준

- JavaScript 구문 검사를 통과한다.
- 데스크톱과 모바일에서 가로 넘침이 없다.
- 기출/예상 및 질문 유형 필터가 동작한다.
- 질문, 답변 패턴, 필러의 추가·수정·삭제가 동작한다.
- 새로고침 뒤에도 localStorage 데이터가 유지된다.
- 예상 질문 카드와 60초 연습 결과에 추천 답변 출처가 표시된다.
- 기존 메모 삭제 키가 유지되고 메모 내용이 추천 답변에 들어가지 않는다.
- 브라우저 콘솔 오류가 없다.
